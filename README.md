# 多平台电商销售数据聚合报表工具

## 项目背景

客户是电商卖家，从多个电商平台导出产品销售数据（Excel），目前**手工**用 VLOOKUP 等操作把品牌/业务员匹配进去，再手动聚合出三张分析报表。每周/每月重复，非常耗时。

## 目标

做一个自动化工具：**输入各平台导出的 Excel → 自动生成带三个 Sheet 的分析报告**，打包成 exe 给客户双击运行。

## 技术栈

- **DuckDB 1.5** — 核心计算引擎（SQL 驱动，零配置，列式存储）
- **Python 3.13** — 胶水层，平台解析，Excel 输出
- **pandas / openpyxl** — 数据处理和 Excel 格式化
- **Streamlit** — 后续做界面（当前阶段只做核心逻辑）
- **PyInstaller** — 后续打包 exe

## 已完成（第一阶段：核心逻辑）

### 项目结构

```
data-arrange/
├── main.py                    # 入口: uv run python main.py <输入> [输出]
├── pyproject.toml
├── config/
│   └── schemas.yaml           # 各平台字段映射配置（平台格式变化只改这里）
├── src/
│   ├── __init__.py
│   ├── ingest.py              # 平台解析器：读Excel → 查映射表 → 统一格式
│   ├── transform.py           # DuckDB 合并清洗：UNION ALL + 推广费归集 + 利润计算
│   ├── reports.py             # 三张报表的 SQL 生成
│   └── export.py              # 格式化 Excel 输出（样式/百分比/冻结/筛选）
├── data/
│   ├── uploads/               # 用户上传文件目录
│   ├── output/                # 报表输出目录
│   └── mappings/              # 映射表（品牌→业务员，可让客户在Excel里编辑）
└── 设计方案.md                # 早期设计文档
```

### 数据流水线

```
各平台导出Excel (.xlsx)
     │
     ▼
① ingest.py 解析                   ← schemas.yaml 定义各平台字段映射
  ├── 帮帮管家婆 → 含品牌/业务员/成本/利润，最完整
  ├── 帮帮旺店通 → 含品牌/业务员（已匹配），无成本
  ├── 1688导出  → 无品牌/业务员，查映射表补齐
  ├── 品牌通导出 → 推广费数据（品牌补贴金额）
  └── 帮帮导出   → 汇总订单
     │
     ▼
② transform.py DuckDB 合并清洗
  ├── UNION ALL BY NAME 合并
  ├── 推广费按品牌+月份聚合，LEFT JOIN 回主表
  ├── 负数数量 → 自动标记退款金额/退货成本
  └── 利润 = 销售额 - 成本总额（或直接用源数据利润字段）
     │
     ▼
③ reports.py 三张报表
  ├── 产品维度: GROUP BY 产品名称,所属月份 → 销售额/成本/毛利/毛利率
  ├── 品牌维度: GROUP BY 业务员,品牌,所属月份 → 含推广费/返点
  └── 业务员周报: LAG() 环比 → 增长率/退货率/投入产出比
     │
     ▼
④ export.py 格式化 Excel
  ├── 表头蓝底白字 + 冻结首行 + 自动筛选
  ├── 百分比列（含"率""比"的字段）自动设置 0.00% 格式
  └── 数字列 #,##0.00 格式
```

### 支持的数据源与字段映射

| 数据源 | 映射配置节 | 说明 |
|---|---|---|
| 帮帮管家婆 | `platforms.帮帮管家婆` | 有品牌/业务员/成本/利润，最完整 |
| 帮帮旺店通 | `platforms.帮帮旺店通` | 有品牌/业务员（已匹配），无成本 |
| 品牌通导出 | `platforms.品牌通平台导出数据` | 推广费来源，有品牌/业务员 |
| 1688导出 | `platforms.1688平台导出数据` | 无品牌/业务员，查 `1688基础数据` 映射 |
| 帮帮导出 | `platforms.帮帮平台导出数据` | 汇总订单，无产品明细 |

### 映射表

| 表名 | 用途 |
|---|---|
| 微信基础数据 | 单位名称 → 品牌 + 业务员 |
| 1688基础数据 | 1688标题 → 自命名品牌 + 业务员 |
| 帮帮基础数据 | 已绑定门店 → 品牌 + 业务员 |
| 品牌通基础数据 | 品牌通渠道 → 现品牌 + 业务员 |

映射表放在 `config/schemas.yaml` 的 `mappings` 节，客户可以直接在 Excel 里编辑。

### 运行命令

```bash
# 单文件模式（一个Excel含多个平台Sheet）
uv run python main.py "数据整理（龙）(1).xlsx" "data/output/报表输出.xlsx"
```

### 当前已处理的细节

- ✅ 日期格式兼容：`2026-05-14` / `20260514` / `2026年05月`
- ✅ 退货识别：负数数量自动标记退款金额和退货成本
- ✅ 推广费归集：品牌补贴按品牌+月份合并
- ✅ 映射表匹配：模糊匹配（1688标题包含关系）
- ✅ 环比计算：DuckDB `LAG() OVER (PARTITION BY ... ORDER BY 所属月份)`
- ✅ 无效月份过滤：未付款订单不参与报表
- ✅ Excel 格式化：表头样式、百分比格式、冻结首行、自动筛选

## 关键设计原则

1. **配置驱动**：字段映射在 YAML 中，格式变化只改配置不改代码
2. **SQL 优先**：核心业务逻辑用 DuckDB SQL，透明可审计
3. **映射表可维护**：客户在 Excel 里直接编辑品牌→业务员对应关系
4. **输出兼容**：生成的 Excel 格式与客户手工报表一致，无缝替换
